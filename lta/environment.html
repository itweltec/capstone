The Environment

A virtual environment configured to emulate an enterprise deployment.

TOOLS

Resources that contributed to environment design and configuration.


Virtualisation Tool • vSphere

VMware's cloud computing virtualized platform.

vSphere was the main tool used to create, configure, store, and connect the virtual environment needed to simulate the scenario. To understand the requirements of each VM and guide this implementation, a topology design was necessary.


Modelling Tool • Diagrams.net

The tool used to create a detailed environment topology.

To gain a better understanding of what the team would actually be creating, a topology diagram was needed to make sense of everything. Diagrams.net was the tool used to create this networking "map" to help the team navigate through the networking side of the project.


Scenario Research • Digital Corpora

The source analysed to inform majority of the scenario.

The project team used Digital Corpora to review what published datasets typically looked like. An especially critical resource from the site was these conference proceedings about creating realistic corpora.

TOPOLOGY




For the topology, an understandable map involving all required virtual machines had to be designed to guide how a believable and secure environment would be created. The topology diagram created to navigate environment implementation is shown above. For network segmentation, port groups were created to allocate VMs to, which essentiallly acted like subnets.

Security systems were then set up to operate both internally and externally so that threats could be addressed inside and outside of the network. A large part of the environment design was having strong defensive measures like this in place to showcase just how they could be bypassed by the right attack and the right attacker, which is the risk real companies must work to mitigate today.

To capture all of the events that took place during the attack, the internal security box was separated in a group with a mirrored port. Because the sccenario involved exfiltration to a third-party, an external network was required as well as an internal one. As shown above, the topology also included the external devices in that network and their details.

Why?

To have a place where an elaborate story could be simulated.

About The Project

Linux Threat Analysis a project undertaken by Team T.Y.P.O., a group of I.T. students looking to contribute up-to-date material for computer forensics education at WelTec. It was based entirely on Linux systems to address a shortage of similar datasets.
