Digital Attendance System

Converging Technology and Business Processes

About The Project

The Digital Attendance System (DAS) is a final year Project undertaken by four students at the Wellington Institute of Technology. The Digital Attendance System was a project that converged existing technology and business processes to form a digital solution that improves and refines the business process.

The root issue addressed by the Project tackled the inconsistent approaches of collecting student attendance at class sessions, which is an important metric for International Students’ Visas. These inconsistent approaches were also error-prone and often time-consuming for Staff Members to both collect and input into the Institute's central attendance system.

Since the System would be used by a range of Staff Members, the Staff interfaces would need to be simple and easy to use while still maintaining all the required functionality.

The Solution

In designing the Digital Attendance System, the team decided to break down the overall System into four Sub-Systems, which would provide the System with a degree of modularity and a separation of concerns. The four Sub-Systems that compose the Digital Attendance System are: a Headless REST API Server, an Android Mobile Application, a Staff Web App, and a Student Web Portal. These four Sub-Systems interact with each other to bring full functionality to the overall System. The User Interfaces were designed to be intuitive and simplistic to use for the users.
The structure of the four Sub-Systems are:

  • Headless REST API Server: This was built with C# / .NET Core and runs on a Linux machine. The Headless API Server is the backbone of the System and maintains the data of the system. The Server’s API Endpoints were protected with JSON Web Tokens.
  • Android Mobile Application: The Android Mobile Application was built using Java and utilises Near Field Communication (NFC) technologies to enable students to record their attendance at class sessions by scanning their Student ID card against a compatible Android device. The Android Application was developed to be offline-first compatible, enabling the Application to record attendance without requiring an active internet connection.
  • Staff Web App: The Staff Web App was built with the Angular framework and allows Staff Members to manage their class sessions, devices connected to the system, and export the attendance data in the CSV format required by the central attendance system that is copied to their clipboard. Google Authentication was integrated into the System to enable Staff Members to use their existing logins.
  • Student Web Portal: The Student Web Portal was also built upon the Angular framework and allows students to record their attendance at class sessions by entering an unique 4-character code that is generated in the Staff Web App and their Student ID number; this then records their attendance at that class session.

The Mobile Application allows the Students' attendance to be recorded by scanning their Student ID Cards, while the Student Portal allows attendance to be collected by Students entering their ID Number. The attendance data is then exported from the System using the Staff Web Application
Together these four Sub-Systems enable ease and simplicity of recording attendance. The System then enables staff to easily export the data into the central attendance system.
