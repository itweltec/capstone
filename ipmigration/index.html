The Problem

SoIT Network Diagram

IP migration is a project of Welling Institute of Technology where the Project team has to meet all the client requirement which are as follows:

1. Simulate existing SoIT public IP network on the packet tracer
2. Migrate new private IP scheme with minimum own time
3. Prepare IP migration plan based on the above exercise.

Currently WelTec B BLock is using the public IP which was claimed through the Historical based resource after Center Institute of Technology merged and formed WelTec. Now that specific IP must be replaced as WelTec has sold it to other parties, it is now mandatory to replace those IP in B Block with new private IP. Therefore they need a ip migration plan and recommendation to minimize

Our client for this project was Jeffry Echano, a Lecturer from WelTec's School of IT, with the responsibility of managing and maintaining the school’s network and infrastructure.

The Solution

We selected hybrid white papers for this project, combination of two internet white papers’ methodology. The maturity of these models and the project environment have more possibility of success rate on IP migration project than other standard project methodologies. Also the project environment is a key factor of project success and that establishes a relationship in between methodology and project success. Our model has 7 steps which are Cleanup, Documentation, Research, Pre-migration Testing, Execution, Post-migration Testing and Turnover.

In Cleaup phase, we gathered the project requirements and generated all the planning documents. During the requirement gathering team analysed and validate existing B-Block network infrastructure and updated network diagram. After systems survey finished, team started simulating the current network on Packet Tracer. Network engineers put their industry experience and academic knowledge to build the entire BBlock network with end to end communication. Then testing team conducted pre-migration test with more than 100 test cases, which was significant phase of this project. Team encountered configuration errors and managed to troubleshoot by producing root cause analysis report.

During the simulation of new IP address migration, project team identified the IP migration procedure with best practises. This was the happening phase where team decided the migration should be done overnight with migration window or step by step without having network downtime. Post-migration testing phase confirmed the network works as before the migration. Which will eliminate users’ support calls for IT support after migration.

Finally, team prepare IP Migration Plan and recommendation documents align with entire exercises.

Templetes and Tools

Cisco Packet Tracer

The Team used Cisco Packet Trace. It is a cross-platform visual simulation tool designed by Cisco that allows any users to create network topologies and imitate modern computer networks by configuration of Cisco routers and switches using a simulated command line interface.

Microsoft Visio

The Team used Microsoft Visio for creating Network Diagram. It is a software for drawing a variety of diagrams which include flowcharts, org charts, building plans, floor plans, data flow diagrams, process flow diagrams, business process modeling, swimlane diagrams, 3D maps, and many more

Google Drive

The team used Google Drive to store files related to the project, allowing for team collaboration and access to files. This project library support throughout the project as central information management system and for document version control.

Discord

Project team used Discord as backup communication platform to conduct team meeting with screen sharing in the event of lockdown period (COVID-19). Moreover, it used to update the development phase team members work progress (follow the sun concept).

Test Cases

The team created Test cases for which contained & described the testing, scopes, and methodology associated with the testing of the project. Team developed 127 testcases templates for pre-migration and post migration testing.

Advance IP scanner

The Team used Advance IP Scanner to find existing IP addresses on the BBlock network and corelated with devices. Team used this tool in the requirement gathering phase to build model network.

Microsoft Project

The project team used Microsoft Project for create project plan. Team used this tool to allocate project resources to respective tasks and find the critical path to apply lag to overcome project milestone delivery late.

Progress report

Project team created this document weekly and presented to advisor. This report explained in detail how you far one've gone towards the completion of a project and as well as forecasting following week project tasks. the weekly team meeting was the place where team used to review and create this document.
