Agile Board Tracker

Bridging the gap between physical and digital

The Problem

In Agile Scrum, the use of a digital and physical scrum board has created significant problems and confusion because there are no controls that guarantee one view or the other. This has provided unnecessary and unproductive work to synchronise the boards; time better spent on other tasks.

The Solution

You can now synchronise your digital Scrum board to match changes made to your physical board, using the Agile Board Tracker. This product allows you to manage your Scrum items on a desktop application, able to be printed and placed upon the physical board. This product also allows for capturing of the physical board, analysing it for changes, and then sending it to the desktop application to update the database with the use of a mobile application.

Development

Agile Scrum

Agile Scrum was utilised to manage our development process. This allowed us to be flexible, and keep the client involved.

Taiga.io

To implement our use of Agile Scrum, we used Taiga. This acted as our digital Scrum board. We also used many of Taiga's project management features.

Discord

The team kept in touch using Discord, by utilising its instant messaging capabilities. It was also used to share images, code and other files.

Google Drive

Drive was used to store files related to the project, allowing for team collaboration and access to files.

GitLab

For storing the code repository, GitLab was used to allow remote access to the latest code through development.

Android

The mobile application was developed for Android devices. This was made easy by the use of the Android Studio IDE.

OpenCV

The image recognition processes were made using OpenCV, a programming library aimed at real-time computer vision.

Barcode Scanning

To identify tasks on a board, we incorporated a simple barcode reader. Barcodes are created by the desktop applcation.
