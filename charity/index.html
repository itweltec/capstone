Context

The Project Client for this project is Clement Swarnappa. Clement works as a Senior Academic Staff Member in the School of IT and Business at Wellington Institute of Technology (WelTec). As the client of this project, he has investigated an opportunity for a student project at a charitable religious organization he volunteers with as the trustee and treasurer. Being the treasurer, he has found the current financial practices tedious, unsecure, and hard to report. He thought, if a web application could do background jobs to prepare and present clearer reporting and include secure financial management it would be excellent. This has led to the current scope of this web application CAPE (Charity Accounting Portal for ENABLE).

Purpose

The purpose of this project was to provide a free scalable solution to religion organisations to manage their financial data in a secure way. The project team created a simple, yet efficient solution to support small scale organisations to manage their accounts securely and accurately. This solution eliminated the use of Excel spreadsheets which can be insecure and tedious. It will also eliminate the use of MYOB and Xero which can be costly.

Methodology and Development

Tools and Methodology Used

Visual Studio 2019

Our project team programmed the project in Visual Studio 2019. The project team used the ASP.NET framework and C# which consists of programming in C#, HTML, CSS and JavaScript.


Github

The project team used GitHub to manage versions and collaborate on the same project. The project team also used GitHub as a backup space.



Scrum

The methodology we used for the project was scrum. This is because we aimed to have weekly/fortnightly meetings with the Client in which where we would show the current progress and give us tasks to do next.

Development Process

Design

Design was the first stage for the project, the project team made three different versions of the front-end design, along with other design document that helped us imagine how the application will look and function.




Development

The project team started with the front-end coding based on the design accepted by the client. Next the project team created the database which stores the data that get inputted. Then the project team coded the forms and reports agreed upon with the client for the application along with a code clean up at the end.

Testing

Before end of the project, the project team did test for the project code. The project team used user acceptance testing and UI testing to make sure project doesn’t contain any bugs or critical errors and is acceptable for the client.

Future

For future projects

The Client has stated in the past that he might continue this project in the future. Our team members recommend a few things that would help that next team who takes on the project next. We suggest that the next team try and experiment with Windows hosting if they want to edit this project which uses ASP.NET cannot use Linux hosting that the Client currently has. The Client uses PHP to write the front-end web pages, we suggest the new team that takes over the project to consider using PHP language for development. The PHP language can use the existing Linux Hosting plan of the client and ensure that there are no problems when linking pages together. The current ASP.NET (C#) code can continue to be edited and modified so the format of the report achieves the client’s satisfaction. Monthly reports and annual reports need to show more information, classification and require a new layout to display all the needed information. SQL databases may need to code and designed from scratch as the current one is a local database and is very basic. Using another software to create the database would be a good idea.
