Goals

The Concept

Kubernetes for WelTec was a project which aimed to deliver a product which could replace Netlab, the School of IT's current lab delivery system, with a Kubernetes-based solution

Netlab is an online learning solution which provides/allows students from the School of IT to access pre-configured virtual lab environments from any internet-capable device. Each lab environment contains one or more virtual computer systems (known as virtual machines) which students can interact with via a web interface in order to complete their assigned lab work.

Our client for this project was Paul Bryant, a Senior Lecturer from WelTec's School of IT, who saw the opportunity to replace Netlab with a more cost-efficent solution after being inspired by a similar system deployed by the Edinburgh Napier University.

Solution

In order to meet our client's need, we decided to develop an ASP.NET Core web application that would allow users to create and interact with lab environments - collections of pre-configured networked Docker containers hosted within a Kubernetes cluster managed by the School of IT.

Although the system which inspired our project was based on Kubernetes, we examined other potential solutions prior to starting development, including OpenStack and Proxmox Virtual Environment. We considered factors such as cost and complexity when making our selection, along with each option's suitability given our clients' needs. In the end, Kubernetes remained our platform of choice, as it appeared to be simpler to implement than OpenStack, the other entirely free option.

Kubernetes is a free and open-source container orchestration platform - it is used to manage container workloads spread across an entire cluster of container hosts, along with ensuring that these workloads can communicate with each other and the wider network. It supports multiple container types, with the most well-known being the Docker container.

Docker containers are an implementation of virtualisation, which allows a 'virtual' computer system to be hosted within another computer system. Containers appear to operate as a separate entity to its host, but still use the host's kernel, bringing efficiency benefits over more mature virtualisation technologies like the virtual machine. In addition, by default, each Docker container is isolated from other containers which share the same host, allowing a single server to host multiple container lab environments.

The final challenge of our application's design was how we would implement user account management for the application. We considered two approaches:

  1. User accounts managed directly through the application.
  2. User accounts managed through an existing authentication and authorisation system controlled by the School of IT
We decided to take the latter approach, integrating our application into the School of IT's existing Google-based email system, as we felt this would be more convenient for users, while also simplifying management of user permissions.

Tools

Visual Studio 2019

Visual Studio was the chosen development environment for the project, as its feature set synergised well with ASP.NET Core, our chosen web application framework, and Microsoft SQL Server, our web application's database solution.

Mockflow WireframePro

The team used the free version of WireframePro to build user interface mock-ups for storyboards.

GitHub

The team used GitHub as the version control solution for the entire project, including development of both the web application and the supporting Kubernetes infrastructure.

Proxmox Virtual Environment (PVE)

PVE is a virtualisation platform which can host virtual machines and LXC containers. Towards the end of application development, a server running PVE hosted a Kubernetes cluster used for system testing.

Minikube

Minikube is a locally-hosted single-node Kubernetes implementation. We used Minikube as our development environment throughout the development process.

yEd Graph Editor

yEd was used by the team to create the overall system diagram.

Microsoft Project

The team used Microsoft Project for our initial project plan. The Gantt chart it generated was a key part of our System Proposal document.

UMLet

UMLet is a free and open-source UML diagramming tool, which was used by the team to produce Use-Case and Activity Diagrams.

End Product

Current Status

As it stands, the Athena application is feature-complete, and could be deployed to a Kubernetes cluster. Currently, all users of the Athena web applications can:

  • create lab environments from a pre-defined template - a set of Kubernetes configuration files which define the resources needed for the lab environment.
  • interact with their lab environments
  • delete their lab environments

On top of this to this basic functionality, School of IT staff can also:

  • create new lab environment templates
  • manage user groups and their memberships
  • limit access to lab environment templates to certain user groups


Further Development of the Application

We were able to finish development of all planned application features, and have completed some testing of the system in operation. Due to complications resuting from the COVID-19 lockdown, we were not able to deploy the system to School of IT hardware for testing at a larger scale.

In addition to the aforementioned testing, there are several avenues for expansion:

  1. Investigate the feasibility of hosting virtual machines inside Docker containers.
  2. Extend the application's user interface so that users can see all their running/current lab environments in one place.
