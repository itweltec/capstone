Just suggesting Basecamp to students isn't enough. We also need to help students and staff learn to use it quickly and easily. To do this we developed 5 sets of instructions - 1 for each major Capstone Project users. These user types are: Advisor, Coordinator, Client, Panel member, and Student.


The Development Process:

Before we could start work on the instructions, we needed to know what to include - what does each user group need to know? We interviewed our Client for information about Capstone Project Advisors, Coordinators, Clients, and Panel members. To learn about the needs of Students we sent a survey to past and present Capstone Project students asking about their experiences with project management tools and what they thought should be included. After reviewing the information, we collected we identified which tasks each user type would need instructions for.

After making the instructions we needed to test them. Our focus during testing was usability - we wanted to know whether users could find the instructions they needed and whether they could follow them successfully. Because Capstone Project staff and students come from a range of backgrounds, we made sure to have both native and non-native English speakers test the instructions. After the first round of testing, we made many improvements to the instruction, and the second round of testing indicated that the instructions were easy to navigate and use.


The Instructions:

The instructions were created as PDFs. They are intended to be used as digital documents, but have been designed so that they can also be used as hard-copies if that is what a user prefers.

The instructions contain many features to help users find what they need easily:

  • A Quick Reference guide with direct links to the most important instructions
  • Hyperlinks to let users move quickly around the document, or back to the Contents
  • A main Contents list, as well as sub-Contents lists for each section
  • A "How to get the most out of these instructions" section
  • Page numbers so the instructions can be used as hard-copy documents
  • Colour coded page banners which allow a user to easily know which section they are in
  • PDFs allow Ctrl+F to be used to search for keywords

We also designed the instructions so that they are easy to follow:

  • Clear, step by step instructions
  • Action focused language e.g. "Click on the To-dos icon"
  • Images which clearly show the user what they need to do for each step
  • Notes with important information or helpful hints

Using these instructions, Capstone Project staff and students can quickly learn how to use Basecamp. This means students are more likely to use Basecamp when it is suggested as a suitable project management system. This will save time for staff by increasing consistency, and help students by removing a potential barrier to their success.
